# Course Registration Wizard 완료 체크리스트

이 체크리스트는 `specify`, `plan`, `tasks`, `implement` 단계에서 요구사항이 누락되지 않았는지 확인하기 위한 문서다.

## 1. Specify 단계

- [ ] `REQUIREMENTS.md`, `AGENTS.md`, `README.md`, `.specify/memory/constitution.md`를 읽었다.
- [ ] `spec.md`에 3단계 사용자 흐름이 포함되어 있다.
- [ ] 개인 신청과 단체 신청이 별도 시나리오로 다뤄진다.
- [ ] 단체 -> 개인 전환 시 확인 후 데이터 초기화 정책이 명시되어 있다.
- [ ] 참가자 이메일 중복 금지 정책이 명시되어 있다.
- [ ] 신청자 이메일과 참가자 이메일 중복 금지 정책이 명시되어 있다.
- [ ] 정원 초과, 마감 강의, 잔여 정원 부족 UX가 명시되어 있다.
- [ ] 강의 목록 로딩, 빈 목록, 조회 실패 상태가 명시되어 있다.
- [ ] 제출 실패 후 입력 데이터 유지와 재시도가 명시되어 있다.
- [ ] 서버 에러 코드 `COURSE_FULL`, `DUPLICATE_ENROLLMENT`, `INVALID_INPUT` 처리가 명시되어 있다.
- [ ] 불필요한 인증, 결제, 관리자 기능이 scope에 들어가지 않았다.

## 2. Plan 단계

- [ ] React + Vite + TypeScript 기반 계획이 수립되어 있다.
- [ ] 폼 상태를 통합 상태로 관리하는 방향이 계획되어 있다.
- [ ] 개인/단체 신청 데이터가 discriminated union으로 설계되어 있다.
- [ ] 유효성 검증 로직이 UI 컴포넌트에서 분리되어 있다.
- [ ] Mock API 전략이 `GET /api/courses`, `POST /api/enrollments` 기준으로 계획되어 있다.
- [ ] 테스트 계약이 `util -> apis/mocks -> hooks -> components -> page` 순서로 설계되어 있다.
- [ ] 선택한 라이브러리의 이유와 대안이 `research.md` 또는 `plan.md`에 기록되어 있다.

## 3. Tasks 단계

- [ ] 각 task가 정확한 파일 경로를 포함한다.
- [ ] 테스트 작성 task가 구현 task보다 앞에 있다.
- [ ] 검증 유틸 테스트 task가 포함되어 있다.
- [ ] 개인/단체 전환 정책 테스트 task가 포함되어 있다.
- [ ] Mock API 성공/실패 응답 테스트 task가 포함되어 있다.
- [ ] 스텝별 컴포넌트 또는 페이지 테스트 task가 포함되어 있다.
- [ ] README 갱신 task가 포함되어 있다.
- [ ] `npm run lint`, `npm run test` 검증 task가 포함되어 있다.

## 4. Implement 단계

- [ ] 1단계에서 강의 카테고리, 목록, 선택, 정원 상태가 동작한다.
- [ ] 2단계에서 공통 필드 검증이 동작한다.
- [ ] 2단계에서 단체 조건부 필드 검증이 동작한다.
- [ ] 3단계에서 전체 요약, 수정 링크, 약관 동의, 제출이 동작한다.
- [ ] 이전 단계로 돌아가도 입력 데이터가 유지된다.
- [ ] blur 시 개별 필드 검증이 동작한다.
- [ ] 스텝 전환 시 현재 스텝만 검증한다.
- [ ] 검증 실패 시 필드별 에러 메시지와 시각적 강조가 제공된다.
- [ ] 가능한 경우 첫 에러 필드로 포커스 또는 스크롤이 이동한다.
- [ ] 제출 중 중복 제출이 방지된다.
- [ ] 제출 실패 후 입력 데이터가 유지되고 재시도할 수 있다.
- [ ] 성공 화면에 신청 번호와 요약 정보가 표시된다.
- [ ] README 필수 항목이 최신 구현 내용으로 갱신되어 있다.

## 5. 최종 검증

가능한 경우 아래 명령을 모두 통과해야 한다.

```bash
npm run lint
npm run test
```

프로젝트 초기화 전이라 명령이 없다면, package scripts를 만든 뒤 다시 검증한다.
